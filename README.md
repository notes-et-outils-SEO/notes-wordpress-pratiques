# 🧱 Notes WordPress pratiques

Ce dossier rassemble des notes simples sur la façon de garder un site WordPress lisible, maintenable et proportionné à son vrai besoin. L'idée n'est pas de transformer chaque projet en machine lourde, ni de faire croire qu'un petit site doit forcément être piloté comme une grosse production éditoriale. Ces textes s'adressent surtout aux personnes qui veulent publier, structurer quelques pages, garder un minimum de méthode et éviter les complications qui arrivent souvent trop tôt. Le fil directeur reste volontairement sobre : choisir moins, documenter un peu mieux, et éviter l'empilement de décisions prises dans la précipitation.

## 🎯 À qui cela peut servir

Ces notes peuvent aider une personne qui lance un site vitrine, un petit blog, une base de contenus modeste ou un projet WordPress sans équipe technique dédiée. Elles peuvent aussi servir à quelqu'un qui reprend un site existant et cherche à comprendre ce qu'il faut garder simple pour ne pas alourdir inutilement la suite. L'objectif n'est pas de couvrir tout l'écosystème WordPress, mais de proposer des repères concrets pour mieux décider sur les points qui créent souvent de la friction : le thème, la structure du site, la documentation minimale et les routines de maintenance.

## 🚫 Ce que ce dossier n'est pas

Ce dossier n'est ni une documentation officielle de WordPress, ni une liste des meilleurs thèmes ou plugins du marché, ni un cadre universel valable pour tous les projets. Il ne remplace pas un audit technique, ne promet pas de performance automatique et ne prétend pas qu'un petit site doit rester figé. Il propose surtout des notes de travail sobres, conçues pour rester utiles quand on veut faire les choses proprement sans se perdre dans un trop-plein d'options.

Le parti pris de l'ensemble est simple : **un site plus léger et mieux compris tient souvent mieux dans le temps qu'un site vite enrichi mais mal documenté**. Ces repères ont été pensés dans cet esprit.
